Callbacks

You can handle asynchronous calls with callback functions.

                
                    get('http://google.ch', function(response, error) {
                        console.log(response);
                    });

                    function get(url, callback) {
                        const data = http.get(url);
                        callback(data);
                    }
This is possible because functions are first class objects.

Callbacks

This can quickly end up in callback hell

                
                    getSomeData(function(data1) {
                        getMoreRelatedData(data, function(data2) {
                            getEvenMoreData(moreData, function(data3) {
                                console.log('got it!');
                            }
                        });
                    });

Promises

Promises make asynchronous calls more readable, especially when chained.

                
                    get(url)
                        .then(d1 => getMoreData(d1))
                        .then(d2 => getEvenMoreData(d2))
                        .then(d3 => console.log(d3));

Promises

                
                    function wait(millis) {
                        return new Promise(resolve => {
                            setTimeout(resolve, millis);
                        });
                    }

Async / Await

When "waiting" for promises to resolve, you can also use async functions to make the code look even more synchronous!

                
                    async function getSomeData(url) {
                        const response = await http.get(url);
                        return response.data;
                    }
